## 🤖 1. In Which Phases Did I Use AI?
During the development of the PatiLink project, I used AI tools (large language models and coding assistants) as supportive aids in the following areas:
* **Database Schema Design**: Modeling C# classes representing the relational structures (One-to-Many, Many-to-Many) of Entity Framework Core models (`UserProfile`, `CareTask`, `Animal`, `LocationPoint`, etc.).
* **Minimal API Endpoint Templates**: Drafting the structure of asynchronous database queries and endpoints (`GET`, `POST`, `PUT`, `DELETE`) protected by JWT authorization in `Program.cs`.
* **BackgroundService Logic**: Building the scheduler mechanism of the `CareStatusBackgroundService` class, which runs continuously on the server side every 15 minutes to monitor feeding timeouts.
* **Leaflet Map Integration**: Setting up the Leaflet map on the React side, rendering radar-animated ping markers (divIcon) dynamically on the map, and populating pop-up data upon clicking markers.

## 🛠️ 3. What Modifications Did I Make to the AI-Generated Code?
AI tools typically produce generic templates. To adapt this code to PatiLink's specific business logic, I made the following modifications:
* **Minimal API Configuration**: I removed the generated traditional Controller classes and integrated all API routing into `Program.cs` in Minimal API format to make the project lightweight and fast.
* **Tailwind v4 Adaptations**: I cleaned up outdated CSS classes and ad-hoc styling generated by the AI, and made the design modern, responsive, and fully compliant with Tailwind CSS v4's `@theme` and custom HSL color definitions.
* **Automated Task Flow**: When the health record API was generated, I personally implemented the logic to "extract the veterinarian's identity (JWT claims) from the request and automatically generate a follow-up task assigned to that veterinarian" and save them under a single transaction.

## 🩺 4. Which Bugs Did I Resolve Personally?
I resolved complex bugs that occurred during development and testing which AI tools were unable to solve:
* **EF Core Entity Tracking Conflict**: I resolved the *'The instance of entity type cannot be tracked'* conflict error that occurred when updating both the task status (`Status = "Completed"`) and the related animal's status (`CareStatus = "Fed"`) upon task completion, by optimizing asynchronous queries and correctly tracking entity states on the DbContext.
* **Camera and Scanner Resource Locking**: I resolved the issue where the QR scanner camera process remained active in the background and locked the device's camera resources even when the scanner modal was closed, by properly stopping camera tracks inside the React `useEffect` cleanup function.
* **Map Render Misalignment (Leaflet Render Bug)**: I resolved the issue where Leaflet container dimensions were not detected correctly upon initial render (causing broken tile images) by programmatically triggering the `map.invalidateSize()` method asynchronously right after the map instance was initialized.

## 🌟 5. What Is the Original Aspect I Contributed to This Project?
The most original aspect of the project that goes beyond the course requirements is the **Location-Verified Smart QR Task System**.

When volunteers take on a feeding or watering task for any animal on campus, they cannot complete the task from home or from an incorrect location. When the volunteer goes to the feeding point and scans the QR code with their camera:
1. The system sends the scanned QR code to the server,
2. The server verifies whether this QR code matches the QR code of the location (`LocationId`) where the animal is currently registered,
3. If the location does not match, a red alert box appears with the message **"Incorrect Location"** and blocks the completion of the task.
4. If the location is correct, the targeted need's card is highlighted in the UI with a glowing yellow frame, helping the volunteer complete the task error-free.

For veterinarians, a bypass mechanism was added allowing them to close medical follow-up tasks directly from the panel without QR scanning. These flows and integrations were designed and implemented entirely based on my own design decisions.
